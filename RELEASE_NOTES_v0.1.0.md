# Release Notes — J-Stack v0.1.0

Release date: 2026-05-30

J-Stack v0.1.0 is the first GitHub release package for the public J-Stack skill system.

## What this release is

A public MIT-licensed Hermes Agent skill collection for agentic journalism, investigation, public writing, evidence discipline, GBrain-backed newsroom memory, and publication-safety workflows.

It contains 26 Markdown-based `SKILL.md` workflows plus references, templates, bundles, validation scripts, manifests, quality reports, red-team hardening reports, and two fictional worked examples.

## What this release is not

J-Stack v0.1.0 is not:

- a certified product;
- a hosted newsroom system;
- legal, compliance, security, tax, or professional advice;
- a guarantee of publication safety;
- a substitute for editorial/legal/security/data-protection review;
- an authorization to publish, contact sources, send messages, file documents, deploy systems, or store sensitive material.

Human approval remains mandatory for consequential actions.

## Highlights

- 26 J-Stack skills across editorial command, research, evidence, legal/ethics, drafting, distribution, ship-gate, post-publication, and GBrain memory workflows.

- Two fictional, non-sensitive worked examples:
  - Data journalism toy dataset workflow.

## Recommended first steps

1. Read `README.md` and `DISCLAIMER.md`.
2. Read `GETTING_STARTED.md`.
3. Run:

```bash
python3 scripts/validate_jstack.py
python3 scripts/hardening_audit_jstack.py
python3 examples/jstack-seo-distribution/metadata-length-check.py
```

4. Pilot only selected skills with synthetic/public material.
5. Keep all real-world publication and external-action decisions behind human review.

## Known limitations

- The repository is a skill/workflow package, not an application.
- Behavior depends on the agent runtime, model, tools, GBrain setup, and user governance.
- US and international publication-risk topics are checklists and escalation prompts, not legal conclusions.
- The examples are intentionally small and synthetic.
- There is no CI workflow configured yet; validation is script-based.
