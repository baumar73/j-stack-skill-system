# Contributing to J-Stack

J-Stack accepts contributions that improve public-interest journalism workflows without weakening evidence discipline.

## Contribution Rules

- Do not add skills that fabricate sources, citations, quotes, or legal certainty.
- Do not add outbound publishing/contact automation as a default behavior.
- Keep GBrain pages concise and non-secret.
- Add tests/validation for every new skill frontmatter.
- Keep the core law visible: no claim ships above evidence grade.

## Review Checklist

- Does the skill have clear trigger/non-trigger sections?
- Does it state GBrain reads/writes/fallback behavior?
- Does it include legal/ethical boundaries if public claims are involved?
- Does it produce concrete artifacts?
- Is it non-overlapping with existing skills?
